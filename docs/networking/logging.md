# Connection Logging and Authentication Telemetry

## Steam credential capture during `SV_DirectConnect`
* `client_t` now tracks the platform credential label, raw token, negotiated result/outcome strings, the most recent status message, and the presented SteamID so the server can describe what happened during each stage of the handshake.【F:src/code/server/server.h†L124-L154】
* `SV_CapturePlatformAuthFromUserinfo` normalises the userinfo keys used by Quake Live (`auth`, `author`, `author2`, and `steamid`) and preserves the results before the VM gets a chance to mutate the string. Each transition (`connect`, `denied`, `accepted`) is reported through `NET_LogAuthTelemetry`, which now includes the backend's `result`, high-level `outcome`, and combined message strings (token summaries plus VM-provided denial text). Even when the join packet lacks a Steam ticket, the telemetry line is tagged with `credential=steam`, ensuring host-side traces clearly identify Steam-aware sessions.【F:src/code/server/sv_client.c†L28-L170】【F:src/code/server/sv_client.c†L563-L595】

## Game-side validation via `ClientConnect`
* `G_RunPlatformAuthChecks` reconstructs the Steam ticket from userinfo, converts it into a `ql_auth_credential_t`, and forwards the request to the platform abstraction (`QL_RequestExternalAuth`). Any rejection bubbles back to the engine as "Failed to verify Steam auth token", matching the Quake Live denial string. Successful validations emit a `SteamAuthAccepted` telemetry line for the server log, while retry-eligible responses surface as `SteamAuthRetry` entries so operators can distinguish transient throttling from hard failures.【F:src/code/game/g_client.c†L37-L205】
* The helper writes the backend's `result`, high-level `outcome`, and most recent message back into userinfo (`ql_auth_result`, `ql_auth_outcome`, `ql_auth_message`) before returning. `SV_FinalisePlatformAuthState` consumes those keys when the VM call completes so `SV_LogPlatformAuth` can forward the exact backend messaging to the shared telemetry stream.【F:src/code/game/g_client.c†L72-L205】【F:src/code/server/sv_client.c†L108-L170】
* `ClientConnect` invokes the helper before bot setup, short-circuiting the join when validation fails and only advertising the `steamid` once the credential passes the external check. When Steam metadata is present, the server prints the numeric 64-bit ID (falling back to the raw string if parsing fails) to match the native logging format.【F:src/code/game/g_client.c†L1032-L1124】

## Shared telemetry emitter
* `NET_LogAuthTelemetry` now records the direction, address, credential label, SteamID, status, backend result, high-level outcome, and a sanitised message in a single log line, giving both the dedicated server and in-process host the same visibility without duplicating string handling.【F:src/code/qcommon/net_chan.c†L744-L804】
